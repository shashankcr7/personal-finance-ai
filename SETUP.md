# SETUP.md — One-time setup before Claude Code builds

Do these in order. Steps marked **[you]** need your login/credentials and can't be automated. Steps marked **[claude code]** are done by the agent once the repo and env exist.

---

## 0. Prerequisites [you]
Install if missing:
- Node.js 18+ (`node --version`)
- Python 3.11+ (`python3 --version`)
- Git, and the GitHub CLI `gh` (optional but easiest)
- VS Code + the **Claude Code** extension (Anthropic publisher)
- Docker Desktop (optional — for a local Postgres during dev)

## 1. Get the scaffold onto your machine [you]
Unzip the provided `finance-ai` folder, then open a terminal in it.

```bash
cd finance-ai
git init
git add .
git commit -m "Initial scaffold: docs, structure, config"
```

## 2. Create the GitHub repo [you]
**With gh CLI (fastest):**
```bash
gh auth login          # one-time, in your browser
gh repo create finance-ai --private --source=. --remote=origin --push
```
**Or manually:** create an empty private repo on github.com named `finance-ai`, then:
```bash
git remote add origin https://github.com/<your-username>/finance-ai.git
git branch -M main
git push -u origin main
```
> The `.gitignore` already excludes `.env` and raw statement files. Confirm `.env` is **not** listed in `git status` before pushing.

## 3. Create the cloud accounts [you]
Sign up (free tiers are fine to start):
- **Supabase** — supabase.com (Postgres + Auth + RLS)
- **Anthropic** — console.anthropic.com (API key for the agent)
- **Vercel** — vercel.com (frontend hosting)
- **Render** — render.com (backend hosting)

You do **not** need a separate SMS/OTP provider — Supabase Auth covers email + 2FA.

## 4. Set up Supabase [you]
1. Create a new project; save the database password.
2. **Project Settings → Database → Connection string:** copy the **Session Pooler** string (**port 5432**). This is required for IPv4 hosts like Render's free tier — the direct connection will silently fail there. Ensure the schema is `public`.
3. **Project Settings → API:** copy the project URL, `anon` key, and `service_role` key.
4. **Authentication → Providers:** enable Email. **Authentication → MFA:** enable TOTP (2FA).
5. You'll add the email allowlist enforcement in app logic (`ALLOWED_EMAILS`).

## 5. Fill in environment variables [you]
```bash
cp .env.example .env
```
Fill each value from the dashboards above. `.env` is git-ignored — never commit it. The backend reads these via `os.environ`; the Claude key stays backend-only and must never appear in frontend code.

## 6. (Optional) Local Postgres for dev [claude code can scaffold]
Developing against a throwaway local DB first avoids risking the real one. A `docker-compose.yml` for local Postgres can be generated by Claude Code; point `DATABASE_URL` at it while building, switch to Supabase when ready.

## 7. Validate your real files first [you] — 10 minutes, do not skip
Before building against assumptions:
- Run one real **CAS** PDF (each of yours) through `casparser` in a throwaway script — confirm both your NSDL/CDSL variants parse.
- Open one **ICICI** and one **Kotak** CSV export — note the exact column names.

These two unknowns drive the schema and the normalizer. If they don't parse cleanly, fix the spec before the agent builds.

## 8. Start Claude Code [you → claude code]
1. Open the `finance-ai` folder in VS Code. Claude Code auto-reads `CLAUDE.md`.
2. Use **Plan mode** for `/backend/parsers`, `/backend/analytics`, `/backend/agent`, auth, and any DB migration. Review those diffs by hand. Auto-accept only for UI scaffolding.
3. First tasks, in order:
   - Scaffold `/frontend` (Next.js + TS) and `/backend` (FastAPI).
   - Write the schema as a **migration file**; review the SQL in Plan mode; run it against **local Postgres** first, then Supabase. Never run a reset/drop casually.
   - Build the **CAS + bank parsers and the analytics functions with tests** — before any UI.
   - Then API → dashboard → agent.

## Guardrails recap (also in CLAUDE.md)
- LLM never does arithmetic; money is `Decimal`/`NUMERIC`, never float.
- Parse-then-delete raw statements; no PAN stored.
- Secrets in env only; Claude key server-side only.
- RLS on every table; allowlist the two emails.
- No buy/sell recommendations — analysis and flagging only.
